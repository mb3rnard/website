The flexible, robust, and web oriented toolbox for Ansible.

A bunch of guidelines to rules them all !

The Manala project was made to use Ansible roles created by Elao to handle web environments and more, it provides configuration examples and best practices to fast build custom-designed infrastructures.

We choosed Ansible as IT automation tool regarding is simplicity and reliability. We are using it to build our infrastructures and we are convinced this is the way to go.

Vagrant is a classic in a developer stack, used to ensure everyone works on the same environment, Vagrant boxes are of course provisioned using Ansible roles.

Docker is the last but not the least, we first used it for test coverage to build CI environment on demand, this last time we are using it for scalability needs of large projects.
